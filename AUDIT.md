# Аудит проекта «Зеркало предков»

Дата первичного аудита: 18 июля 2026.

## Исходное состояние

- Проект остаётся на Vue 3 + Vite + JavaScript, MediaPipe Tasks Vision и Canvas 2D.
- Машина состояний `IDLE → LIVE → PROCESSING → RESULT` сохранена.
- Pose, Gesture и Image Segmenter используют локальные модели и WASM.
- Исходный прогон: `npm test` — 11/11 тестов успешно; `npm run build` — успешно.
- Репозиторий Git в переданной папке отсутствует, поэтому сравнение выполняется по файлам и тестам.
- Визуальная проверка живой камеры на целевом MacBook из текущей среды пока не выполнена.

## Подтверждённые проблемы

### P0 — неправильный fallback подвижного рукава

Сценарий: `cherkeska`, `hybrid` или `dynamic`, одна кисть/локоть временно теряется.

- Ожидание: рукав мягко возвращается в исходное статичное положение.
- Фактически: слой рукава деформируется полной сеткой корпуса. Узкий фрагмент PNG может растянуться через весь костюм и создать разрыв.
- Затронуты: `src/core/renderer.js`, `src/core/costumeLayout.js`.
- Исправление: отдельная статичная сетка рукава, рассчитанная из исходных координат рукава и текущей сетки корпуса.

### P0 — отсутствие ограничений валидности mesh

Сценарий: рука быстро проходит перед грудью, локоть шумит или на кадр меняется геометрия сустава.

- Ожидание: ткань остаётся непрерывной, без перевёрнутых треугольников.
- Фактически: renderer проверяет только вырожденный исходный треугольник; площадь и ориентация целевого треугольника не контролируются.
- Риск: вспышки, переворот текстуры, длинные клинья.
- Исправление: безопасные ограничения длины конечности, ширины соседних рядов и проверка target mesh с возвратом к статичной сетке.

### P1 — ассеты касаются границ PNG

Автоматическая проверка alpha показала непрозрачные граничные пиксели:

- `cherkeska.png`: 268;
- `burka.png`: 111;
- `gabali_green.png`: 88;
- `gabali_red.png`: 318.

Для текущей задачи изменения будут ориентированы на мужские `cherkeska` и `burka`. Касание границы повышает риск обрезания и тёмной каймы при affine-деформации. Нужен прозрачный запас с переносом всех anchors/fit-координат.

### P1 — неполное освобождение ресурсов

`stopMirror()` останавливает MediaStream, но не отменяет сохранённый animation frame и не закрывает MediaPipe-задачи. При повторном mount/HMR возможны накопленные ресурсы.

### P1 — недостаточная диагностика производительности

Debug показывает общий FPS, но не показывает Pose inference, Gesture inference, render frame и причину блокировки фото. Без этого нельзя честно локализовать падение ниже 20 FPS.

### P2 — отказ одного необязательного ассета ломает всю загрузку

`Promise.all` завершает запуск ошибкой при отсутствии любого костюма, убора или фона. Для киоска нужен гарантированный fallback на черкеску и доступные кнопки.

## Риски изменений

- Добавление padding меняет систему координат PNG; одновременно должны переноситься anchors, polygons и rows.
- Слишком жёсткие ограничения рукавов дадут заметную задержку или отрыв от кисти.
- Изменение зеркалирования без живой камеры может инвертировать handedness; текущая логика жеста не меняется без MacBook-проверки.
- Нельзя считать 20 FPS подтверждёнными без запуска на демонстрационном MacBook.

## План небольших проверяемых исправлений

1. Добавить статичную сетку рукава и unit-тест потери кисти.
2. Добавить валидатор деформационной сетки и безопасный fallback.
3. Добавить `scripts/audit-assets.mjs`, прозрачный padding и перенос координат мужских ассетов.
4. Добавить performance/debug-метрики без записи landmarks в Vue каждый кадр.
5. Закрывать MediaPipe-задачи, MediaStream и rAF при остановке.
6. Проверить photo pipeline и безопасную деградацию сегментации.
7. Проверить UI на доступных автоматизированных размерах, затем выполнить ручную проверку камеры на MacBook.
8. После каждой итерации запускать связанные тесты; в конце — полный `npm test` и `npm run build`.

## Способ проверки

- `node scripts/audit-assets.mjs`
- `node scripts/render-fit-guides.mjs`
- `npm test`
- `npm run build`
- Ручная калибровка: `?debug=1&fit=hybrid`, затем `dynamic` и `fixed`.

## Результат выполненной итерации

- Исправлен fallback рукавов: отдельная статичная сетка вместо body mesh.
- Добавлены ограничения выбросов суставов и проверка winding/площади треугольников.
- `cherkeska` и `burka` получили прозрачный запас 16 px с идемпотентным переносом опор.
- Исправлен `fixed`: шестирядная сетка и посадка папахи внутри 16:9.
- Добавлены безопасная загрузка ассетов, fallback черкески и режим без необязательного убора.
- Добавлены закрытие MediaPipe-задач, остановка rAF/MediaStream и countdown timer.
- Добавлены performance-метрики, причина блокировки фото, кнопка входа и retry загрузки.
- Главная страница, manifest, Pose/Gesture/Segmenter-модели и WASM проверены локально: HTTP 200.
- После реализации: 16 автоматических тестов и production build проходят.
- Не подтверждено из текущей среды: реальный FPS, камера, сегментация живого человека и визуальные viewport-проверки. Нужен финальный прогон на демонстрационном MacBook.
