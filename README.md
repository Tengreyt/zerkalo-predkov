# Зеркало предков

Интерактивная виртуальная примерочная национального костюма — экспонат стенда
«Экспонаты Республики» цифрового музея ЧГУ им. А.А. Кадырова.
Хакатон «Ночь в музее», 17–19 июля 2026.

Всё работает on-device в браузере: фото посетителей никуда не отправляются.

## Запуск

```bash
npm install
npm run dev        # http://localhost:5180
npm test           # unit-регрессии геометрии и жестов
npm run audit:assets # размеры, alpha, границы и отсутствующие PNG
npm run build      # статика в dist/ для Vercel / GitHub Pages
```

Демо: MacBook, Chrome, полноэкранный режим (⌘⇧F), разрешить камеру.

## Управление жестами (без пульта и клавиатуры)

Распознавание руки — MediaPipe **GestureRecognizer** (тот же движок и WASM, что и
Pose Landmarker; модель `public/models/gesture_recognizer.task`, локально, без CDN).
GestureRecognizer работает на CPU с фиксированным интервалом около 12 Hz, пока
Pose остаётся на GPU. Низкоуверенный результат дополнительно проверяется по
геометрии 21 точки кисти. Краткие потери руки не обрывают удержание, а после
смены экрана требуется опустить руку — один жест не срабатывает дважды.
Команда фото принимает только поднятую раскрытую ладонь лицевой стороной к
камере: проверяются размер, высота, разворот плоскости и handedness. При потере
ладони таймер удержания замораживается и не может завершить съёмку сам.

| Жест | IDLE | LIVE (примерка) | RESULT |
|------|------|-----------------|--------|
| ✋ раскрытая ладонь (удержать ~0.9 c) | начать примерку | сделать фото (запуск 3-2-1) | ещё раз |
| 👍 палец вверх | — | следующий костюм | ещё раз |
| ✌️ victory | начать примерку | предыдущий костюм | — |
| ☝️ указательный вверх | — | развернуть/свернуть описание | — |
| ✊ кулак | — | показать/скрыть головной убор | — |

Логика: `core/gestureRecognizer.js` (ML + геометрический fallback) → `core/gestureController.js`
(дебаунс, cooldown, удержание с прогрессом 0..1) → маппинг жест→экран в
`composables/useMirror.js` (`dispatchGesture`). Пороги/тайминги — в
`settings.js → gesture`. Экранная легенда и кольцо удержания — `GestureHud.vue`.

В IDLE основной вход — жест ✋; авто-вход по присутствию оставлен страховкой
(`gesture.idleAutoEnterMs`, по умолчанию 6 c), чтобы киоск не «завис».
Все жесты продублированы кнопками на экране (мышь/тач).

## Реальные ассеты костюмов

Фото костюмов лежат в `assets_src/` (исходники), а скрипт
`node scripts/prepare-real-assets.mjs` готовит из них кадры для наложения:
отдельно обрабатывает шахматный и градиентный фон, не «выедая» белые вставки
платьев, срезает голову/убор манекена, пишет PNG в `public/assets/costumes/`
и проставляет якоря плеч в `manifest.json`. Папаха вырезается из фото черкески
отдельным головным убором.

Готовые костюмы: `cherkeska`, `burka` (черкеска с буркой), `gabali_green`,
`gabali_red`. Для черкески используется объёмный AR-убор `papakha_ar.png`,
подготовленный по предоставленному референсу. `gown` (мантия) пока
заглушка — ждём фото. Якоря/срезы подстраиваются долями в самом скрипте;
после замены исходников проверить наложение через `?debug=1` (метки якорей).

Если придёт официальный `zerkalo_predkov_assets_v2.zip` с готовыми прозрачными
PNG и своим `manifest.json` — распаковать поверх `public/assets/costumes/`,
скрипт подготовки тогда не нужен.

Тексты карточек — `src/config/descriptions.js` (ключи = id костюмов),
при необходимости заменить текстами из `OPISANIYA.md`.

Мюридская форма НЕ реализована — ждём референсы от университета.

## Калибровка комплекции

Открыть `http://localhost:5180/?debug=1` — на экране живое значение `r`,
текущий класс и fps. Прогнать 10–15 человек, записать их `r`, выставить пороги
`THIN` / `LARGE` в `src/config/settings.js` → `bodyClass`.

Класс считается по медиане `r` за 15 кадров во время таймера 3-2-1.
Класс выбирает файл (форму одежды), ширина плеч в пикселях — только масштаб.

## План Б («волшебное зеркало»)

По умолчанию используется `hybrid`: корпус следует за плечами/бёдрами, подол
стабилизирован относительно оси тела, рукава следуют за руками. Быстрые режимы:

- `?fit=hybrid` — основной музейный режим;
- `?fit=dynamic` — максимум движения по всем точкам;
- `?fit=fixed` — полностью статичная рамка («волшебное зеркало»).

В `fixed` поза остаётся триггером присутствия. Рамка настраивается в
`fixedFrame`; это безопасный режим для защиты при сложном освещении.

## Мультирежим

По умолчанию `peopleMode: 'single'` — костюм на самом крупном (ближайшем)
скелете. Для до 3 человек: `peopleMode: 'multi'` — пресет «микс слева направо»
от выбранного в карусели костюма.

## Архитектура

```
src/
  config/settings.js        все пороги, тайминги, флаги стратегий
  config/descriptions.js    тексты карточек костюмов
  store/appState.js         машина состояний IDLE→LIVE→PROCESSING→RESULT + автосброс 30 c
  composables/useMirror.js  оркестратор: камера → поза → отрисовка → снимок
  core/
    assets.js               manifest.json + предзагрузка изображений
    poseTracker.js          MediaPipe Pose Landmarker (numPoses: 3, WASM локально)
    gestureRecognizer.js    MediaPipe GestureRecognizer (жесты руки, локально)
    gestureController.js    дебаунс/cooldown/удержание жестов → редкие события
    smoothing.js            адаптивный One Euro filter (стабильность без сильной задержки)
    bodyClassifier.js       r = shoulder*0.6 + hip*0.4, медиана за 15 кадров
    costumeLayout.js        hybrid | dynamic | fixed + сетки корпуса/рукавов/убора
    renderer.js             Canvas 2D: видео + кусочно-аффинная деформация PNG
    photoProcessor.js       стоп-кадр: сегментация → фон по костюму → цветокоррекция
```

MediaPipe WASM и модели лежат локально (`public/mediapipe/`, `public/models/`)
— никаких CDN, киоск не зависит от внешней сети. Сегментация запускается
только на стоп-кадре, в live-превью работает один Pose Landmarker.

## Точная посадка одежды

В `dynamic`-режиме костюм больше не рисуется одним жёстким прямоугольником.
`costumeLayout.js` строит сетку из шести рядов: верх PNG, плечи, талия, бёдра,
колени и подол у лодыжек. `renderer.js` переносит текстуру по треугольникам
Canvas 2D, поэтому одежда следует за наклоном корпуса, шириной бёдер и длиной
человека, сохраняя исходный крой и альфа-канал PNG.

Общие параметры находятся в `settings.js → overlay.bodyWarp`. Положения
смысловых рядов задаются долями исходного PNG в `sourceRows`; если официальный
архив будет иметь другой крой, достаточно откалибровать эти четыре числа без
изменения математики. Для точной сетки должны быть видны плечи, бёдра, колени и
лодыжки. При частичной позе костюм плавно скрывается и появляется подсказка.

У костюмов с `fit.sleeves` исходный PNG при первом использовании разделяется на
корпус и два рукава. Каждый рукав имеет три смысловые линии в координатах
исходника и пять рядов деформации по цепочке плечо → локоть → кисть. Если кисть
потеряна, рукав автоматически возвращается в статичную посадку на корпусе.

Головные уборы с `fit.mode: "head-mesh"` получают четыре отдельные линии
деформации: верх, край верхнего эллипса, начало декоративной ленты и низ.
Масштаб и наклон считаются по ушам и глазам, нос задаёт мягкую перспективу при
повороте, а под нижним краем рисуется контактная тень. Параметры конкретного
убора (`width_factor`, `height_factor`, `bottom_lift`) находятся рядом с его
якорями в `manifest.json`.

В `?debug=1` поверх одежды показываются подписанные линии плеч, талии, бёдер,
коленей, подола и обоих рукавов. Это основной режим калибровки новых PNG.
Команда `node scripts/render-fit-guides.mjs` создаёт изображения с исходными
масками и опорами в `tmp/fit-guides/`, чтобы проверить разрез слоёв без камеры.

`prepare-real-assets.mjs` добавляет мужским `cherkeska` и `burka` прозрачный
запас 16 px. Вместе с PNG переносятся anchors, polygons и sleeve rows; повторный
запуск идемпотентен и не накапливает сдвиг. Renderer отклоняет перевёрнутые или
вырожденные mesh-треугольники. При потере кисти/локтя рукав получает отдельную
статичную сетку в своей исходной области, а не растягивается сеткой всего тела.

Debug-панель раз в секунду показывает среднее время Pose, Gesture, Canvas render
и общего frame, а после снимка — время photo pipeline и сегментации. Частый цикл
не записывает landmarks в Vue. Реальные 20 FPS подтверждаются только прогоном на
целевом MacBook.

## Безопасная обработка фото

Сегментация получает только чистый кадр камеры. Костюм хранится на отдельной
прозрачной канве и добавляется после вырезания человека, поэтому ML больше не
может разрезать одежду на фрагменты. Категориальная маска дополняется узкой
страховочной маской по скелету (голова, корпус, руки, ноги), которая закрывает
внутренние дыры, и получает мягкое перо 1.5 px. При ошибке сегментатора система
оставляет исходный кадр целиком вместо показа повреждённого результата.

При размонтировании приложение отменяет `requestAnimationFrame`, останавливает
MediaStream и закрывает Pose Landmarker, Gesture Recognizer и Image Segmenter.
Повреждение необязательного PNG исключает только этот костюм; отсутствие убора
оставляет рабочий костюм без него. `cherkeska.png` — обязательный fallback.

## QR

QR ведёт на реальный снимок посетителя, а не на страницу проекта. Передача
выполняется только после явного нажатия «QR — забрать на телефон»:

1. JPEG отправляется в **private Vercel Blob** через `POST /api/photos`.
2. Генерируется случайная ссылка на `photo.html?id=...`.
3. На телефоне показываются фото и кнопка «Скачать фото».
4. Через один час `GET /api/photo` перестаёт выдавать файл.
5. Истёкшие blobs физически удаляются при следующих загрузках.

До нажатия QR фотография остаётся только на MacBook. Локальный Vite не запускает
Vercel Functions, поэтому для полной проверки использовать Vercel Preview либо
`vercel dev`.

### Подключение QR-хранилища на Vercel

1. Открыть проект в Vercel → **Storage** → **Create Database** → **Blob**.
2. Выбрать режим **Private**.
3. Подключить Blob store к проекту. Vercel автоматически добавит секрет
   `BLOB_READ_WRITE_TOKEN`.
4. Повторно развернуть проект.
5. Сделать фото, нажать QR и проверить ссылку с телефона через мобильный интернет.

Без `BLOB_READ_WRITE_TOKEN` интерфейс показывает честную ошибку и сохраняет
локальную кнопку «Скачать здесь»; фальшивый QR больше не создаётся.
